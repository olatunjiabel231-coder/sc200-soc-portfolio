## 2. Detection Engineering – Brute Force Login Detection — Completed

In this project I created a custom Microsoft Sentinel analytics rule to detect brute-force login attempts.

The rule monitors **Azure AD sign-in logs** and triggers an alert when:

* 10 or more failed login attempts occur
* from the same IP address
* targeting a user account
* within a 1-hour time window

This project demonstrates detection engineering using **KQL queries and Sentinel analytics rules**.

Documentation:
`PROJECT-1-detection-engineering/detection-rule/README.md`

---

## 3. Security Incident Investigation — Completed

After the detection rule generated alerts, I conducted a full SOC-style investigation.

The investigation included:

* Reviewing the incident queue
* Analyzing correlated alerts
* Examining the attack story graph
* Investigating the source IP entity
* Reviewing query results generated by the detection rule

The investigation confirmed multiple failed authentication attempts from a single IP address targeting one user account, indicating a **brute-force login attempt**.

Documentation:
`PROJECT-1-detection-engineering/incident-investigation/README.md`
